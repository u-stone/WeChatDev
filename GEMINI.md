# Project Overview: WeChat Mini Game + C++ WebAssembly

This project demonstrates how to integrate C++ code (compiled to WebAssembly) into a WeChat Mini Game. It features a 2D particle physics simulation where gravity, boundary collisions, and circle-circle collisions are handled in C++, while the JavaScript side manages rendering via Canvas 2D and the animation loop.

## Project Structure

- **`cpp-module/`**: C++ Sub-project (Physics Engine → WebAssembly)
  - `src/math/`: Header-only 2D vector library (`vec2.h`).
  - `src/physics/`: Core physics logic (`particle.h`, `world.h`, `world.cpp`).
  - `src/demo.cpp`: WASM export layer using `EMSCRIPTEN_KEEPALIVE`.
  - `CMakeLists.txt`: Build configuration for CMake + Emscripten.
  - `build.sh`: One-click build script for Debug and Release targets.
- **`minigame/`**: WeChat Mini Game Sub-project
  - `game.js`: Entry point, animation loop, and Canvas 2D rendering.
  - `js/wx-polyfills.js`: **Critical** compatibility layer for `WXWebAssembly`.
  - `js/wasm-loader.js`: Loads Wasm modules and binds C functions using `cwrap`.
  - `wasm/`: Build artifacts (`demo.js`, `demo.wasm`) generated by `cpp-module/build.sh`.
- **`docs/`**: Supplemental documentation.

## Building and Running

### 1. Build C++ to WebAssembly
Requires [emsdk (Emscripten SDK)](https://emscripten.org/docs/getting_started/downloads.html) to be installed and activated in the current shell.

**macOS / Linux:**
```bash
cd cpp-module
./build.sh debug    # Includes sourcemaps for C++ debugging
./build.sh release  # Optimized for performance and size
```

**Windows (PowerShell):**
```powershell
cd cpp-module
.\build.ps1 debug   # Includes sourcemaps for C++ debugging
.\build.ps1 release # Optimized for performance and size
```

### 2. Run the Mini Game
1. Open **WeChat DevTools**.
2. Import the project using the `minigame/` directory as the root.
3. Use "Tourist Mode" or a valid AppID to run the simulation in the simulator.

## Development Conventions

- **C++ Coding Style**: Adheres to Google C++ Style Guide (PascalCase for classes, camelCase for methods).
- **Wasm Interop**: 
  - All exported C++ functions must be wrapped in `extern "C"` and annotated with `EMSCRIPTEN_KEEPALIVE`.
  - JS side uses `cwrap` to bridge types. Supported types are `number` (int/float/pointer), `string`, and `null` (void).
- **Language**: All comments and documentation are in English.
- **Encoding**: All files must be saved in UTF-8.
- **Formatting**: Lines should not exceed 100 characters.

## Key Implementation Details

- **Memory Management**: The Wasm module uses a shared memory buffer. The `wasm-loader.js` handles initialization and mapping.
- **WeChat Compatibility**: `wx-polyfills.js` is required to be the first import in `game.js`. it maps `WXWebAssembly` to `WebAssembly` and provides necessary fetch polyfills for local file access.
- **Debugging**: C++ source debugging is supported in WeChat DevTools using the `.wasm.map` generated in `debug` builds.
